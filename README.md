# Coffeebara

Coffeebara는 같은 저장소 안에서 계속 이어지는 프로젝트입니다.

초기 단계의 v1은 Kakao Local API 기반의 카페 검색, 지도 탐색, 선호하는 카페 등록, 추천 지향 가정을 중심으로 출발했습니다.
그 단계는 지금도 중요한 이정표이며, 이 저장소 안에 아카이브와 기록으로 남아 있습니다.

지금의 방향은 더 조용하고 더 오래 남는 쪽으로 옮겨가고 있습니다.

**Coffeebara는 개인 커피 기록과 카페 맥락을 함께 다루는 프로젝트로 다시 정리되고 있습니다.**

이 프로젝트가 지향하는 것은 단순한 카페 탐색이 아니라, 시간이 지나도 다시 꺼내 볼 수 있는 커피 경험의 기록입니다.

- 원두
- 브루잉 파라미터
- 테이스팅 노트
- 주관적 인상
- 어떤 카페에서 샀는지
- 어떤 카페에서 마셨는지
- 시간에 따라 쌓이는 개인 취향 경향

이 프로젝트는 공개 추천 플랫폼이 아니라, 카페 맥락을 함께 남기는 개인 커피 로그북 쪽에 더 가깝습니다.

이전 단계에서 정리한 방향 메모는 [product-direction-notes.md](./product-direction-notes.md)에 남아 있고, 현재 작업 원칙과 코드 해석 기준은 [AGENTS.md](./AGENTS.md)에 정리되어 있습니다.

## 현재 상태

이 저장소는 방향이 바뀌었다고 해서 리셋된 프로젝트가 아닙니다.
다만 현재 런타임 기준으로는 이미 꽤 많은 분리가 진행된 상태입니다.

지금의 코드베이스를 해석할 때 중요한 점은 아래와 같습니다.

- v1의 탐색 UI와 일부 상태 구조는 여전히 이어져 있습니다
- 하지만 카페 DB 적재 흐름은 현재 활성 기능으로 보지 않습니다
- 현재 런타임은 카페 검색, 지도 조회, 계정 흐름, 프론트 UI 정리에 더 가깝습니다
- 즉, 단순한 디커플링 메모 단계가 아니라 실제 분리와 제거가 어느 정도 진행된 상태입니다

## 현재 실제로 동작의 중심에 있는 것

### Backend

- Spring Boot 4.0.5 + Java 21 + Gradle
- MariaDB + MyBatis 기반 애플리케이션 구조
- Kakao OAuth 로그인과 사용자 계정 처리
- `GET /api/auth/status`
- `POST /api/auth/guest`
- `POST /api/auth/logout`
- `PATCH /api/auth/profile/display-name`
- `GET /api/cafes/search`
- `GET /api/cafes/{placeId}`
- `GET /api/cafes/map`
- `GET /api/cafes/map/search`
- 검색/지도 조회용 캐시와 rate limit

### Frontend

- Next.js 기반 UI
- 로그인 화면
- 헤더 + 사이드바 + 메인 컨텐츠 레이아웃
- Kakao Map 기반 탐색 화면
- 키워드 검색과 지도 범위 기반 조회
- locale 선택 UI
- 저장 카페 목록과 상세 화면
- 저장 카페 관련 상태를 로컬 스토리지 중심으로 관리하는 흐름

## 현재 활성 기능으로 보지 않는 것

문서상 과장 없이 정리하면, 아래는 현재 메인 기능으로 소개하지 않는 편이 맞습니다.

- 카페 DB 적재
- 추천 엔진
- 브루 기록 CRUD
- 원두 기록 CRUD
- 개인 아카이브 완성형 기능

특히 카페 적재는 예전 구조의 흔적이 일부 남아 있어도, 현재 상태를 설명할 때는 **활성 기능**으로 쓰지 않는 편이 맞습니다.

## v1와 현재의 관계

이 저장소는 v1을 버린 것이 아니라 문서와 구조 안에 남겨 둔 채 계속 가고 있습니다.

정리하면 아래와 같습니다.

- v1은 카페 탐색과 선호 카페 축적 중심의 단계였습니다
- 현재는 추천 가정을 뒤로 물리고, 개인 기록 쪽으로 방향을 다시 잡고 있습니다
- 검색과 지도 같은 자산은 계속 살아 있습니다
- 다만 카페 적재를 중심에 두던 해석은 현재 코드 상태와 맞지 않습니다

즉, 같은 저장소를 계속 사용하지만, 현재 런타임의 중심은 이미 달라졌습니다.

## 앞으로의 제품 방향

현재 제품 방향은 아래에 가깝습니다.

- 원두 기록
- 브루 기록
- 카페 맥락
- 테이스팅 노트
- 만족도와 무드
- 개인 히스토리

Kakao 자산도 여전히 중요하지만 역할이 바뀌었습니다.

- Kakao 로그인은 개인 계정 식별과 연속성을 돕습니다
- Kakao Map은 카페를 찾고 선택하는 데 쓰입니다
- 카페 데이터는 기록에 연결될 맥락 정보로 다뤄집니다

더 이상 추천 후보를 대규모로 적재하는 중심 수단으로 설명하지 않습니다.

## 실행 메모

### Backend

저장소 루트에서 실행:

```bash
./gradlew bootRun
```

### Frontend

`frontend/` 안에서 실행:

```bash
cd frontend
npm install
npm run dev
```

## 로컬 실행 시 참고

로컬 실행에는 최소한 아래 설정이 필요합니다.

- Kakao REST API 키
- Kakao JavaScript 키
- 데이터베이스 설정

현재 카페 적재는 비활성화된 상태로 보더라도, 계정 처리와 애플리케이션 구조에는 여전히 DB가 사용됩니다.

## 정리

Coffeebara는 이미 한 번 만들어 둔 카페 탐색 프로젝트를 버리고 새로 시작하는 저장소가 아닙니다.
같은 코드베이스를 유지하면서, 현재 상태에 맞게 해석과 방향을 다시 정리하고 있는 프로젝트입니다.

지금의 런타임 중심은 카페 검색, 지도 조회, 계정 흐름, UI 기반 정리에 있습니다.
그리고 그 위에서 앞으로는 원두, 브루, 노트, 무드, 카페 맥락을 함께 남길 수 있는 개인 커피 아카이브 방향으로 이어지게 됩니다.
