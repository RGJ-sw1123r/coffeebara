# Coffeebara 제품 방향 메모

## 1. 처음에 두었던 가정

- Kakao Local API + 추가 처리 + LLM 조합이면 개인 취향 기반 카페 추천까지 자연스럽게 이어질 수 있다고 봤다.
- 선호 카페와 그 이유를 모으면 사용자 취향 프로필을 만들 수 있다고 봤다.

## 2. 구현과 검토 과정에서 확인한 점

- 사용자 선호(reason)는 어느 정도 구조화할 수 있다.
- Kakao Local API는 장소 탐색과 수집에는 충분하다.
- 다만 후보 카페를 비교하고 추천하기에 필요한 item-side 데이터는 충분히 확보되지 않았다.

## 3. 제품 방향에서 조정이 필요했던 부분

- 사용자 데이터만으로 추천까지 자연스럽게 이어질 것이라는 가정은 강했다.
- Kakao 장소 데이터만으로 카페의 성격을 안정적으로 비교하기에는 정보 밀도가 낮았다.
- 크롤링, 스크래핑, LLM은 보강 수단이 될 수 있지만 이 문제를 단독으로 해결하는 기반은 아니었다.

## 4. 현재 시점에서 더 현실적인 방향

- 강한 추천 서비스보다 개인용 카페 탐색 및 저장 도구에 더 가깝다.
- 카페 검색, 지도 탐색, 카페 저장, 선호 이유 기록, 태그/메모 같은 축은 여전히 유효하다.
- 추천 기능은 더 많은 item-side 데이터 확보 전략이 준비된 뒤 다시 정의하는 편이 적절하다.

## 5. 이 저장소에 남은 핵심 자산

- Kakao 기반 카페 검색
- 지도 탐색 UX
- 카페 저장과 DB 적재 흐름
- 로그인/Auth 확장 여지를 고려한 기본 구조
- AI 협업 과정과 의사결정 기록

## 6. 정리

이 메모는 프로젝트를 부정적으로 정리하기 위한 문서가 아니라, 구현 결과를 바탕으로 어떤 방향이 검증되었고 어떤 방향은 추가 조건이 필요했는지 남겨 두기 위한 기록이다.
